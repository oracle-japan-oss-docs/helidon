モジュール io.helidon.config.objectmapping
パッケージ io.helidon.config.objectmapping

注釈インタフェース一時

@Retention(RUNTIME) @Target({METHOD,FIELD,CONSTRUCTOR}) public @interface Transient
JavaBeanの直列化復元サポートからJavaBeanのプロパティ、メソッドまたはコンストラクタを除外するために使用される注釈。 注釈は、JavaBeanプロパティpublic setter、publicプロパティ・フィールド、publicコンストラクタ、またはpublic builderおよびbuildメソッドで使用できます。

注釈は、Valueと一緒に同じJavaBeanプロパティに適用できません。

次の例では、timestamp構成値が使用可能であっても、プロパティtimestampは設定されません。 プロパティtimestampは、直列化復元プロセスによって完全に無視されます。 

 public class AppConfig { private Instant timestamp; private String greeting; @Transient public void setTimestamp(Instant timestamp) { // <1> this.timestamp = timestamp; } public void setGreeting(String greeting) { // <2> this.greeting = greeting; } //... }
  1. 直列化復元中にsetTimestamp(Instant)メソッドがコールされることはありません。
  2. 一方、setGreeting(String)は、greeting構成値が使用可能な場合にコールできます。
構成例: 
 { "app" : { "greeting" : "Hello", "timestamp" : "2007-12-03T10:15:30.00Z" } }
app構成ノードをAppConfigインスタンスとして取得しています: 
 AppConfig appConfig = config.get("app").as(AppConfig.class); assert appConfig.getTimestamp() == null;
